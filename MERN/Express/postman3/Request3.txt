                    <main role="main" class="article-content">
                        <p>Since we have now walked through Postman and seen <a
                                href="https://toolsqa.com/postman/create-new-request-in-postman/"><em><strong>How to Create and Save a new Request in postman</strong></em></a>,
                            it's time to get our hands on the first <em><strong>GET Request in Postman</strong></em>.
                            When we request from a client machine (<em>User</em>) to a server machine, we follow an
                            architecture and HTTP Protocol. I suggest you go through the below tutorials to establish a
                            nice understanding of <em><strong>HTTP Protocol, Request &amp; Response</strong></em>. These
                            can be viewed here:</p>
                        <ul>
                            <li><a
                                    href="https://toolsqa.com/client-server/client-server-architecture-and-http-protocol/"><em><strong>Client Server Architecture and HTTP Protocol</strong></em></a>
                            </li>
                            <li><a
                                    href="https://toolsqa.com/client-server/http-request/"><em><strong>HTTP Request</strong></em></a>
                            </li>
                            <li><a
                                    href="https://toolsqa.com/client-server/http-response/"><em><strong>HTTP Response</strong></em></a>
                            </li>
                        </ul>
                        <p>Assuming you are now familiar with the HTTP protocols and architecture, we will now talk
                            about one specific type of request which is a <em><strong>GET</strong></em> request. A
                            <em>GET</em> request is used to get the information from the server and does not have any
                            side effects on the server.
                            <em>Side-effects mean there is no updation/deletion/addition of data on the server when you are making this type of request, you just request from the server and the server responds to the request.</em>
                        </p>
                        <p>A <em>GET</em> request has all its information inside the <em><strong>URL</strong></em>, and
                            since URL is visible all the time, it is advisable not to use this type of request while you
                            send some sensitive information such as passwords.
                            <em><strong>For example</strong>, when you press search after writing anything in the search box of google.com, you actually go for a GET request because there is no sensitive information and you are just requesting the page with search results, you notice the same search string in URL.</em>
                        </p>
                        <p>You may also go through the recording of the Postman Tutorial where our experts have
                            explained the concepts in depth.
                        <div class="embed-responsive embed-responsive-16by9"><iframe
                                class="embed-responsive-item youtube-player" type="text/html" width="640" height="390"
                                src="https://www.youtube.com/embed/87C2TLdCqL4" frameborder="0" webkitallowfullscreen
                                mozallowfullscreen allowfullscreen></iframe></div>
                        </p>
                        <p>In this image, as you can see, there is a drop-down button which has different types of
                            request types according to the API need. As of now do not worry about all of these different
                            <a
                                href="https://toolsqa.com/client-server/http-request/"><em><strong>HTTP Requests</strong></em></a>,
                            as we will cover each of these in this Postman Tutorial series. But for now, just focus on
                            the GET Request.</p>
                        <ol>
                            <li>Select <em><strong>GET</strong></em> from the list of request types.</li>
                        </ol>
                        <p><img src="/gallery/Postman/1.GET%20Request%20in%20Postman.png" alt="GET Request in Postman">
                        </p>
                        <ol start="2">
                            <li>Enter <em><strong>www.google.com</strong></em> in the address bar as written in the
                                above image and <em>Press <strong>Send.</strong></em> Now, look at the
                                <em><strong>Status Code.</strong></em></li>
                        </ol>
                        <p><img src="/gallery/Postman/2.Status_Code_200.png" alt="Status_Code_200"></p>
                        <p>Different status codes have different meanings and it does not matter whether it is a GEt
                            request or any other type of request. In this scenario, we have status code
                            <em><strong>200 OK</strong></em> which means that EndPoint is correct and it has returned
                            the desired results. We will show some more status codes later.</p>
                        <p>The colorful text inside the box below is the Response from the server. If you observe
                            closely inside the response box you will see the page code has been sent to us. The above
                            tab says <em><strong>Body</strong></em>. Body means you have selected to view the response
                            body which is been shown inside the box. In Body, you will see three options.</p>
                        <p><img src="/gallery/Postman/3.Pretty_Raw_Preview.png" alt="Pretty_Raw_Preview"></p>
                        <ul>
                            <li><em><strong>Pretty:</strong> In this code will be shown in a colorful manner with different keywords colored differently and will be indented for some of the formats for good reading.</em>
                            </li>
                            <li><em><strong>Raw:</strong> Same as pretty part with no colors and in single lines.</em>
                            </li>
                            <li><em><strong>Preview:</strong> This shows the preview of the page that has been sent. Don't worry about the google doodle if it has not been loaded properly. Try any other website by yourself.</em>
                            </li>
                        </ul>
                        <p>A <em><strong>response</strong></em> is a more detailed topic than it needs to be explained
                            in this chapter. We will be explaining the response completely in the next chapter.</p>
